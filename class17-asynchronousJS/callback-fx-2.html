Let's break down this JavaScript code step by step, focusing on how callbacks work and what the code does.

Overview

This code demonstrates the use of a higher-order function removeNeg that takes an array and a callback function as arguments. The callback determines which elements from the input array are kept in the output array. Three different callbacks are used to filter the array in different ways:

  1. (x) => x >= 0 (keep positive numbers and zero),
  2. (x) => x < 0 (keep negative numbers),
  3. isPrime (keep prime numbers).

JavaScript Code

1. The Array

javascript
const myNumbers = [4, 1, -20, -7, 5, 9, -6, 3, 8, -9, 0, -1];
  • This is the input array containing a mix of positive, negative, and zero values.

2. The isPrime Function

javascript
function isPrime(num) { if (num <= 1) return false; // Numbers ≤ 1 are not prime if (num === 2) return true; // 2 is prime if (num % 2 === 0) return false; // Even numbers > 2 are not prime for (let i = 3; i <= Math.sqrt(num); i += 2) { if (num % i === 0) return false; // If divisible by any odd number, not prime } return true; // Otherwise, it's prime }
  • This function checks if a number is prime.
  • A prime number is a number greater than 1 that is divisible only by 1 and itself.
  • It uses an efficient approach by:
    • Ruling out numbers ≤ 1.
    • Handling 2 as a special case (the only even prime).
    • Checking divisibility only up to the square root of the number (using Math.sqrt).
  • Examples from myNumbers:
    • isPrime(5) → true (prime).
    • isPrime(4) → false (divisible by 2).
    • isPrime(-7) → false (negative numbers aren’t prime).

3. The removeNeg Function (Higher-Order Function)

javascript
function removeNeg(numbers, callback) { const myArray = []; for (let i = 0; i < numbers.length; i++) { if (callback(numbers[i])) { myArray.push(numbers[i]); } } return myArray; }
  • This is the key function that uses a callback.
  • Parameters:
    • numbers: The input array (e.g., myNumbers).
    • callback: A function that takes a single number and returns true or false.
  • How it works:
    • It creates an empty array myArray.
    • It loops through each element in numbers.
    • For each element, it calls the callback function with that element as an argument.
    • If the callback returns true, the element is added to myArray.
    • Finally, it returns myArray.
  • This function is generic—it doesn’t care what the callback does, as long as it returns a boolean.

4. Using removeNeg with Callbacks

javascript
const posNumbers = removeNeg(myNumbers, (x) => x >= 0); // Keep numbers ≥ 0 const negNumbers = removeNeg(myNumbers, (x) => x < 0); // Keep numbers < 0 const primeNumbers = removeNeg(myNumbers, isPrime); // Keep prime numbers
  • Here, removeNeg is called three times with different callbacks:
    1. (x) => x >= 0:
      • An arrow function that returns true if the number is non-negative.
      • Result: [4, 1, 5, 9, 3, 8, 0].
    2. (x) => x < 0:
      • An arrow function that returns true if the number is negative.
      • Result: [-20, -7, -6, -9, -1].
    3. isPrime:
      • The isPrime function defined earlier.
      • Result: [5, 3] (only 5 and 3 are prime in the array; negative numbers and non-primes are excluded).

What Are Callbacks?

  • A callback is a function passed as an argument to another function, which is then executed inside that function.
  • In this code:
    • (x) => x >= 0, (x) => x < 0, and isPrime are callbacks.
    • removeNeg uses the callback to decide which numbers to keep.
  • This is a powerful pattern in JavaScript because it makes removeNeg reusable for any filtering logic, not just removing negatives.

How It All Fits Together

  1. The removeNeg function iterates over myNumbers.
  2. For each number, it calls the provided callback:
    • If the callback returns true, the number is kept.
    • If false, it’s skipped.
  3. Three different callbacks filter the array in three different ways:
    • Positive numbers (including zero).
    • Negative numbers.
    • Prime numbers.

Example Execution

For myNumbers = [4, 1, -20, -7, 5, 9, -6, 3, 8, -9, 0, -1]:

  • posNumbers:

  • primeNumbers:
    • isPrime(4) → false, skip.
    • isPrime(5) → true, keep.

    • Result: [5, 3].

Key Takeaways

  • Callbacks make the code flexible—you can filter the array however you want by passing different functions.
  • The removeNeg function isn’t just for negatives—it’s a general-purpose filter.
  • The isPrime function adds an interesting twist by filtering based on a mathematical property.
